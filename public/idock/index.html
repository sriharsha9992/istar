idock

a multithreaded virtual screening tool for flexible ligand docking

Jobs

New job

The input to idock is threefold:

  • A receptor in PDBQT format. The receptor can contain one or more chains, which should start with ATOM records and end with a TER record. Refer to tutorial 1 on how to prepare a receptor in PDBQT format with MGLTools.
  • A search space defined by a center and a size in unit Å. The search space should define a binding site (e.g. enzymatic site or allosteric site) of the receptor in cuboid shape. Refer to tutorial 2 on how to define a binding site with MGLTools, Chimera or PyMOL.
  • A set of ligands to dock. Totally 12,171,187 ligands are collected from ZINC. Filtering conditions on 9 molecular properties (e.g. mwt, logp, hbd, hba, nrb) can be specified as closed intervals. Only the ligands satisfying all the 9 filtering conditions will be docked.

Our workstations execute idock 2.1 and perform two-phase docking:

  • In phase 1, the selected ligands will be docked in a coarse but fast manner.
  • In phase 2, the top 1000 ligands will be docked again in a fine but slow manner.

As result, you can download:

  • phase1.csv.gz: the predicted free energy, RF-Score, and 9 molecular properties of the selected ligands produced in phase 1.
  • phase2.csv.gz: the predicted free energy, RF-Score, and 9 molecular properties, as well as link to substance information and supplier list of the top 1000 ligands produced in phase 2.
  • hits.pdbqt.gz: the predicted conformations of the top 1000 ligands produced in phase 2.

Number of ligands satisfying all the 9 filtering conditions: 1,107

ZINC ligands

12,171,187 ligands are collected from version 2012-04-06 of the All Clean subset of ZINC and converted into PDBQT format. We thank Professor John J. Irwin, the developer and maintainer of ZINC, for granting us permission to use ZINC with three conditions:

  • We shall provide links to http://zinc.docking.org/substance/zincid for top hits so that users can seek for the most current purchasing information at ZINC's official web site.
  • We shall limit the number of top hits for download to 1000 ligands from a single job.
  • We shall update our ligands when ZINC data is updated so that users can benefit from the most current ligand data.

RF-Score

RF-Score is a machine learning approach to predicting protein–ligand binding affinity. It has been integrated into the idock daemon as an alternative option for rescoring predicted conformations. RF-Score is licensed under CC BY-SA 3.0. We thank Dr. Pedro J. Ballester for sharing with us the code and documentation of RF-Score.

Tutorials

Tutorial 1: prepare a receptor in PDBQT format with MGLTools

Download a receptor at PDB. Take HIV-1 reverse transcriptase (2ZD1) as an example.

Retain only the "ATOM" or "HETATM" records of necessary chain(s) where the binding site of interest resides. In 2ZD1.pdb, the ligand TMC278 binds at the allosteric site, which resides in chain A. Chain A atoms start at line 613 and end at line 5118 with the "TER" record. Save lines 613 - 5118 to a new file 2ZD1rec.pdb.

Download MGLTools at http://mgltools.scripps.edu/downloads and install it.

Use the prepare_receptor4.py script to convert a receptor into PDBQT format.

  • The -r 2ZD1rec.pdb option specifies a receptor for input.
  • The -o 2ZD1rec.pdbqt option specifies a receptor for output.
  • The -A hydrogens option adds both polar and non-polar hydrogens.
  • The -U lps_water_deleteAltB option removes lone pairs, water residues, and XX@B atoms.

On Windows

cd C:\Program Files (x86)\MGLTools-1.5.6rc3
python Lib\site-packages\AutoDockTools\Utilities24\prepare_receptor4.py -r 2ZD1rec.pdb -o 2ZD1rec.pdbqt -A hydrogens -U lps_waters_deleteAltB

On Linux

cd ~/mgltools_x86_64Linux2_1.5.6rc3
pythonsh MGLToolsPckgs\AutoDockTools\Utilities24\prepare_receptor4.py -r 2ZD1rec.pdb -o 2ZD1rec.pdbqt -A hydrogens -U lps_waters_deleteAltB

Copy the content of 2ZD1rec.pdbqt and paste it into Receptor in PDBQT

Tutorial 2: define a binding site with MGLTools, Chimera or PyMOL

A binding site can be defined by:

  • MGLTools
  • Chimera
  • PyMOL

MGLTools

Save the ligand TMC278 (lines 8533 - 8560) to a new file 2ZD1lig.pdb.

Use the prepare_ligand4.py script to convert a ligand into PDBQT format.

  • The -l 2ZD1lig.pdb option specifies a ligand for input.
  • The -o 2ZD1lig.pdbqt option specifies a ligand for output.
  • The -A hydrogens option adds both polar and non-polar hydrogens.
  • The -U '' option removes nothing, i.e. retains lone pairs and non-polar hydrogens.

On Windows

cd C:\Program Files (x86)\MGLTools-1.5.6rc3
python Lib\site-packages\AutoDockTools\Utilities24\prepare_ligand4.py -l 2ZD1lig.pdb -o 2ZD1lig.pdbqt -A hydrogens -U ''

On Linux

cd ~/mgltools_x86_64Linux2_1.5.6rc3
pythonsh MGLToolsPckgs\AutoDockTools\Utilities24\prepare_ligand4.py -l 2ZD1lig.pdb -o 2ZD1lig.pdbqt -A hydrogens -U ''

Run AutoDockTools, which is a component of MGLTools. From the menu of ADT4.2

  • Grid -> Macromolecule -> Open -> 2ZD1rec.pdbqt. Ignore all the warnings about charge, because idock ignores charge.
  • Grid -> Set Map Types -> Open Ligand -> 2ZD1lig.pdbqt.
  • Grid -> Grid Box.

The "Grid Options" dialog pops out.

  • From the menu, click Center -> Center on ligand.
  • Scroll "Spacing (angstrom)" from 0.375 to 1.000.
  • Manually adjust number of points in x-, y-, z- dimension and x y z center, so that the grid box fully covers the ligand.

Paste the corresponding values into center_x, center_y, center_z, size_x, size_y, size_z.

Chimera

Download Chimera at http://www.cgl.ucsf.edu/chimera/download.html#daily and install it. Note that the AutoDock Vina plugin is only available in daily builds. It is not available in Chimera 1.6.2 or earlier versions.

File -> Fetch by ID. By default PDB is chosen. Enter 2ZD1 and click Fetch.

Tools -> Surface/Binding Analysis -> AutoDock Vina. Check "Resize search volume using". Use button 2 to manually adjust the grid box.

PyMOL

Download PyMOL at http://pymol.org and install it. An educational version is available at no cost for educational use only.

Download the AutoDock plugin at https://github.com/Pymol-Scripts/Pymol-script-repo/raw/master/plugins/autodock_plugin.py

Plugin -> Manage Plugins -> Install... -> Choose autodock_plugin.py. Restart PyMOL. Note that the AutoDock plugin can only be installed by PyMOL 1.2r1. It is not supported by PyMOL 1.3r1 or later versions.

Plugin -> PDB Loader Service -> Enter 2ZD1. Plugin -> Autodock/Vina -> Grid Settings. Set Spacing to 1. Manually adjust X-, Y-, Z-points and Grid Center Coordinates.
